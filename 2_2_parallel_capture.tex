\subsection{Parallel Capture and Cross-Shot Deadline Contention}
\label{sec:parallel-capture}

Without parallel capture, the application accepts the next shutter input only after the preceding capture has collected its required frames and metadata and completed the Draft Sequence.
Parallel capture removes this interface-level serialization.
The hardware abstraction layer (HAL) can report \texttt{captureAvailable} while buffers and Draft work from an earlier capture remain outstanding, allowing the application to initiate the next capture sooner and improving shot-to-shot responsiveness.

However, every accepted request starts an independent Capture Timeout clock, so parallel capture creates multiple overlapping deadlines.
The requests can overlap, but their Draft Sequences still run one at a time in capture order.
A later capture may therefore wait behind earlier Draft work while its own timeout clock continues to run.
This queueing delay can cause a timeout even when the later capture's own processing time is unchanged, as illustrated in Figure~\ref{fig:overlap}.
