\begin{abstract}
We present $\ourtool$, a practical technique that refines previously solved query results into reusable data, enabling efficient symbolic execution across successive software releases. Existing symbolic execution techniques repeatedly spend testing effort re-solving path conditions encountered during earlier-version testing, even though adjacent releases typically share most of their code. While reusing previously solved query results appears promising, naive query reuse suffers from high storage and matching costs and frequently regenerates test cases that have already been produced in previous releases. To address these challenges, $\ourtool$ separately refines satisfiable and unsatisfiable query results, generalizes the refined query data into reusable representations, and efficiently reuses them during symbolic execution of later software releases, progressively reducing SMT solver invocations while avoiding regenerating the same test cases across releases. We implemented $\ourtool$ on top of KLEE and evaluated it across consecutive releases of 15 open-source C programs, spanning roughly seven years of software evolution. Experimental results show that $\ourtool$ consistently outperforms both standard symbolic execution and naive query-reuse baselines, increasing branch coverage by 40.2\% on average across releases. Furthermore, $\ourtool$ detected 22 unique bugs, including five that were missed by all competing approaches.
