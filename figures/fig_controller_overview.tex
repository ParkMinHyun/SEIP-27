  \caption{Architecture of the Context-Aware Draft Sequence Controller
  inside the camera framework, between the application and the HAL. Draft
  stages (colors as in Figure~\ref{fig:overlap}) execute top-down on the
  single Draft worker; double-headed arrows mark admission-gated optional
  stages, single-headed arrows measured-only work. The Execution Profiler
  classifies each node into a workload key and mediates between the
  pipeline and the controller: \ding{183}~an optional stage runs only if
  the calibrated bound $U(\sigma)$ of its remaining suffix fits the live
  budget $b$; \ding{184}~admitted stages run under a watchdog that
  reserves the mandatory tail; \ding{185}~observed durations and sequence
  residuals train the shared model at capture end. On the arrival side,
  the framework's performance-policy layer consults Capture-Availability
  Pacing before delivering each \texttt{captureAvailable} callback,
  \ding{182}~releasing it to the application only after the model-derived
  delay $\delta$, which gates when the next shot can start.}
